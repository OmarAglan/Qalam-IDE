# تكامل قلم مع تكوين

قلم لا يحلل `مشروع.تكوين` ولا رسائل أدوات البناء البشرية. يستهلك عقدين يملكهما
تكوين:

- `تكوين أهداف --جسون` يعيد `takween-targets-v1` لاختيار هدف قابل للبناء أو
  التشغيل أو الاختبار.
- أوامر البناء تستقبل `--ملف_أحداث <مسار>`، ويقرأ قلم
  `takween-build-events-v1` كسطور JSON أثناء التنفيذ.

## مسار العملية

1. يجد `BuildManager` أقرب جذر يحوي `مشروع.تكوين`.
2. يطلب فهرس الأهداف من تكوين ويصفيه حسب capability؛ لا يستنتج النوع من البيان.
3. يمرر الأمر العربي القياسي واسم الهدف المختار كوسيط مستقل.
4. ينشئ مسار JSONL مؤقتا باسم Unicode، ويمرره عبر `--ملف_أحداث`.
5. يقرأ `ProcessWorker` السطور المكتملة فقط، ويحفظ السطر الجزئي حتى يكتمل أو
   تنتهي العملية.
6. يتحقق `BuildManager` من schema ونوع كل حقل وتسلسل يبدأ من واحد واسم العملية
   ووجود `operation_finished` ومطابقة كوده لكود process.
7. تتحول الأحداث المقبولة إلى تقدم عربي في شريط الحالة. تبقى stdout/stderr في
   الطرفية للبشر ولا تستخدم لتصنيف النجاح.

أي schema مجهول أو سطر تالف أو فجوة تسلسل أو نهاية مفقودة يجعل النتيجة فشل
process محليا، حتى لو كان خرج النص يبدو ناجحا.

## الإلغاء

يطلب `Shift+F5` أو أمر لوحة الأوامر «إلغاء العملية الحالية» إيقاف process.
يميز قلم النتيجة المحلية بكود `-2` ومعرف `TOOLING_CANCELLED`. عند طلب المستخدم
للإلغاء يسمح عقد v1 بغياب `operation_finished` ولا يحوله إلى خرق بروتوكول.
يختبر fixture مستقل إنهاء process طويل على Windows وLinux. امتلاك شجرة العمليات
كاملة عبر Windows Job/POSIX process group يبقى تقوية لاحقة قبل تشغيل lifecycle
لحزم خارجية غير موثوقة.

## التوافق

يتجاهل parser الحقول الإضافية، لكنه لا يقبل تغيير نوع حقل مطلوب أو schema آخر.
تظل تشخيصات Baa السريعة عبر `diagnostics-json-v1` مستقلة عن قناة أحداث تكوين.
